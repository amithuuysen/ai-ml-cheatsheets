Machine Learning Paradigms — Architecture Diagrams

1. Supervised Learning Flow

Labeled Data
(X, y)
→
Model
(learns mapping)
→
Predictions
(ŷ)
↓
Calculate Error
(Loss = y - ŷ)
↓
Update Parameters
(Gradient Descent)
↻ Iterate until convergence

Key Characteristics:

  • Teacher present: Training data includes correct answers (labels)
  • Goal: Learn function f: X → y that generalizes to unseen data
  • Feedback: Direct error signal from comparing predictions to true labels
  • Examples: Classification (cat/dog), Regression (house prices)

2. Unsupervised Learning Flow

Unlabeled Data
(X only)
→
Model
(finds structure)
→
Discovered Patterns
(clusters/features)
↓
Grouping / Transformation
(segments, dimensions)
↓
Evaluate Intrinsically
(silhouette, reconstruction)

Key Characteristics:

  • No teacher: No labels or correct answers provided
  • Goal: Discover hidden structure, patterns, or representations in data
  • Feedback: Intrinsic quality metrics (no ground truth)
  • Examples: Customer segmentation, anomaly detection, dimensionality reduction

3. Reinforcement Learning Loop

Agent
(makes decisions)
→
Action
(at)
→
Environment
(state st)
↓
Reward Signal
(rt)
+
New State
(st+1)
↓
Update Policy
(maximize Σ rewards)
↻ Continuous learning loop
⟲

Key Characteristics:

  • Interactive learning: Agent learns by interacting with environment
  • Goal: Find optimal policy π that maximizes cumulative reward
  • Feedback: Delayed reward signals (not immediate labels)
  • Trade-off: Exploration (try new actions) vs. Exploitation (use known good actions)
  • Examples: Game playing, robotics, autonomous driving, resource allocation
