Dimensionality Reduction: 4×4 → 2×2 (75% fewer parameters) 2. Translation Invariance: Small shifts in input don't affect output 3. Feature Retention: Keeps strongest activations (most important features) 4. Computational Efficiency: Faster training and inference Output Size Formula: [(W - K) / S] + 1 W=input size, K=pool size, S=stride Other pooling types: Average Pooling (takes average), Global Pooling (entire feature map → single value)
